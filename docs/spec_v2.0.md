# 📘《图像缩放 / 缩放+旋转 FPGA 项目规范说明书 v2.0》

> 本版在 v1.0 基础上重构为 **两阶段落地方案**：
>
> - **Stage A：先完成“纯缩放”硬件链路**
> - **Stage B：在 Stage A 稳定基础上扩展到“同时支持缩放和旋转”**
>
> 目标不是一次性做全，而是先做出 **可跑通、可综合、可验证** 的缩放平台，再把几何变换扩展成统一仿射变换平台。

---

# 1. 项目目标与边界

## 1.1 项目名称

- 实时图像几何变换加速器设计
- 推荐工程名：`image_geo_accel`

## 1.2 平台与工具链

- FPGA 开发板：PYNQ-Z2
- 目标主频：`100 MHz`
- 外部存储：PS 侧 DDR
- 开发语言：`SystemVerilog`
- 工具链：Vivado

## 1.3 输入输出约束

- 输入图像最大尺寸：`7200 × 7200`
- 输出图像最大尺寸：`600 × 600`
- 当前默认图像格式：**8-bit 灰度图**（1 byte/pixel）
- 输入图像位于 DDR
- 输出图像写回 DDR

## 1.4 功能分阶段目标

### Stage A：纯缩放

实现一个只支持缩放的硬件加速链路：

- 输入：DDR 中原图
- 输出：DDR 中缩放结果
- 只处理缩放，不处理旋转
- 优先完成最近邻，再扩展双线性

### Stage B：缩放 + 旋转统一支持

在 Stage A 稳定后，扩展为统一几何变换平台：

- 同时支持缩放与旋转
- 旋转角度范围：`0° ~ 90°`
- 内部采用**统一逆映射坐标框架**
- 先支持最近邻，再扩展双线性

# 2. 关键设计结论（必须统一认识）

## 2.1 本项目必须分两步做

严禁一开始就直接实现“大图 DDR 输入 + 任意缩放 + 任意旋转 + 双线性 + 高性能 AXI”。

必须遵循：

1. **先完成缩放版本**，打通完整工程链路
2. **再扩展缩放+旋转版本**，复用已有平台层

## 2.2 缩放和旋转的正确关系

当同时支持缩放和旋转时，工程最终目标不是“两个独立模块串联”，而是：

> 用一个统一的逆映射框架完成几何变换。

即对每个输出像素坐标 `(x_out, y_out)`，反算其在输入图中的源坐标 `(x_src, y_src)`，再进行取样/插值。

## 2.3 边界处理规则固定

统一采用以下规则：

> **若逆映射后的源坐标越界，则输出背景值 0。**

说明：

- 背景值固定为 `8'h00`
- 不做镜像延拓
- 不做边缘复制
- 不动态扩展输出画幅

## 2.4 输出画幅策略

本项目输出画幅受控，最大不超过 `600×600`。

因此：

- **不采用旋转后自适应扩画幅方案**
- 默认由软件/控制寄存器给定 `dst_w`、`dst_h`
- 硬件只在给定输出画幅内逐点生成像素

## 2.5 旋转中的三角函数不是主瓶颈

当引入旋转时，`sinθ` / `cosθ` 不是逐像素实时计算。

统一约定：

- `sinθ` / `cosθ` 由软件预先计算并写入寄存器
- 硬件内部只做**定点乘加 / 递推更新**

真正的性能难点是：

- DDR 访存组织
- 局部缓存
- 插值
- AXI 吞吐率

## 2.6 旋转阶段不能简单依赖“少量行缓存”

对于纯缩放，行缓存是合理主方案；
对于缩放+旋转，源坐标访问变成二维斜向访问，单纯两行/四行缓存不能从根本上解决 DDR 访问碎片问题。

因此统一策略为：

- **Stage A（纯缩放）**：以行缓存为主
- **Stage B（缩放+旋转）**：优先采用“先缩小，再旋转”的两级架构

这是本项目当前推荐的工程落地路线。

# 3. 总体技术路线

## 3.1 Stage A：纯缩放架构

```text
DDR (PS)
   ↓
AXI Read Engine
   ↓
Source Row Fetch / Line Buffer
   ↓
Scale Core
   ├── nearest
   └── bilinear
   ↓
Row Pack / Writeback Buffer
   ↓
AXI Write Engine
   ↓
DDR (PS)
```

### Stage A 设计原则

- 大图始终放在 DDR 中
- PL 只做局部缓存，不缓存整帧原图
- 按输出图逐行生成结果
- 优先保证功能正确，再优化 burst 和时序

## 3.2 Stage B：缩放+旋转推荐架构

**推荐采用“两级处理”**：

### 推荐路线（优先实现）

```text
原图 DDR
   ↓
Stage B-1: 预缩放（规则缩放）
   ↓
中间小图（<= 600×600 或控制范围内）
   ↓
Stage B-2: 旋转 / 缩放+旋转几何变换
   ↓
输出 DDR
```

### 采用此路线的原因

1. 大幅降低旋转阶段的数据量
2. 旋转阶段更容易使用片上缓存或小范围缓存
3. 避免直接对 5000×5000、7200×7200 大图做碎片化随机访问
4. 更适合当前板卡、频率和实现难度

### 非优先路线（暂不作为首版目标）

```text
大图 DDR
  ↓
统一仿射逆映射
  ↓
直接随机读原图四邻域
  ↓
插值
  ↓
输出
```

此路线保留作为后续性能研究方向，但**不是当前 Codex 首版实现目标**。

# 5. 算法定义与固定规则

## 5.1 坐标系约定

统一采用图像像素坐标系：

- 左上角为 `(0, 0)`
- `x` 向右增加
- `y` 向下增加

## 5.2 缩放统一采用逆映射

对输出像素 `(x_out, y_out)`：

```text
x_src = x_out * scale_x
y_src = y_out * scale_y
```

其中：

```text
scale_x = src_w / dst_w
scale_y = src_h / dst_h
```

说明：

- 这里是数学意义上的比例
- RTL 中必须使用定点实现
- 禁止使用浮点 IP 作为首版依赖

## 5.3 旋转统一采用逆映射

当引入旋转时，对输出点先平移到输出中心坐标系，再通过逆旋转映射回源图坐标。

统一原则：

- 绕图像中心旋转
- `sinθ` / `cosθ` 由软件计算后写寄存器
- RTL 内部使用定点数表示

## 5.4 插值支持顺序

### Stage A

- 必做：最近邻
- 必做：双线性
- 不做：双三次

### Stage B

- 必做：最近邻
- 必做：双线性
- 不做：双三次

> 说明：双三次已从当前目标中移除，不纳入 v2.0 执行范围。

## 5.5 双线性边界合法性

双线性需要 4 点：

- `(x_int, y_int)`
- `(x_int+1, y_int)`
- `(x_int, y_int+1)`
- `(x_int+1, y_int+1)`

因此，合法插值区域统一判定为：

```text
0 <= x_int < src_w - 1
0 <= y_int < src_h - 1
```

若不满足，则走边界分支，输出 0。

# 6. 访存与缓存策略

## 6.1 总原则

- 输入整帧放 DDR
- 输出整帧放 DDR
- PL 只做局部缓存
- 首版不追求“零重复访存”的最优方案
- 先实现正确可控的缓存策略

## 6.2 Stage A 缓存策略（纯缩放）

### 最近邻

- 可按需读取源行
- 至少支持 1 行缓存

### 双线性

- 至少支持 2 行缓存
- 推荐按输出行处理时预取需要的两条源行

## 6.3 Stage B 缓存策略（缩放+旋转）

统一要求：

- **不要把“多加几条行缓存”视为根本解法**
- 首版优先走“预缩放得到小图，再旋转”路线

对旋转阶段：

- 小图可以整体缓存到片上，则优先整体缓存
- 若片上资源不足，则采用小范围 tile / block 缓存
- 不要求首版实现复杂 cache replacement 策略

## 6.4 允许的工程折中

若平台资源和实现复杂度限制较大，允许：

- 先将大图缩放到较小中间图
- 再对中间图做旋转
- 即使带来两次重采样，也优先保证系统落地

此折中为**本项目推荐工程策略**，不是缺陷。

# 7. AXI 与平台层规范

## 7.1 分层原则

### 平台层负责

- AXI 读写
- 地址生成
- burst 管理
- 局部 buffer 管理
- 任务调度
- 状态机控制

### 算法层负责

- 坐标计算
- 像素取样
- 插值计算
- 边界判断

### 强制规则

```text
算法模块禁止直接访问 AXI
AXI 模块禁止实现算法逻辑
```

## 7.2 AXI 使用规则

- 使用 INCR burst
- burst 不得跨 4KB 边界
- 必须检查 `RRESP / BRESP`
- 必须处理 `RLAST / WLAST`
- 首版优先稳定正确，不追求最激进 outstanding 深度

## 7.3 AXI 模块职责建议命名

```text
ddr_read_engine
ddr_write_engine
scaler_ctrl
geo_ctrl
```

说明：

- 不再强绑定 `line_reader` 这种名称到所有阶段
- Stage B 可能不是单纯“行读取”模式

# 11. RTL 编码规范

## 11.1 基本规则

- 使用 `SystemVerilog`
- 时序逻辑使用 `always_ff`
- 组合逻辑使用 `always_comb`
- 不在核心路径中引入不必要的 `real` / `shortreal`

## 11.2 赋值规则

| 类型 | 赋值 |
|---|---|
| 时序 | `<=` |
| 组合 | `=` |

## 11.3 reset 规范

- 默认同步复位
- reset 仅作用于关键控制寄存器
- 不对大数组做清零 reset

## 11.4 FSM 规范

统一使用 `typedef enum logic`。

例如：

```systemverilog
typedef enum logic [2:0] {
    S_IDLE,
    S_PREP,
    S_FETCH,
    S_RUN,
    S_DONE
} state_t;
```

## 11.5 定点实现规范

- 所有比例、坐标、sin/cos 一律使用定点表示
- 推荐统一引入 `FRAC_W` 参数
- 首版可采用如 `Qm.n` 风格，例如 `Q16.16` 或按模块具体精简
- 禁止算法核内部偷偷混入浮点计算

# 13. 性能与实现优先级

## 13.1 优先级排序

统一按以下顺序优化：

1. **功能正确**
2. **仿真通过**
3. **可综合**
4. **时序收敛**
5. **性能优化**

禁止一开始为了追求吞吐率而牺牲结构清晰度。

## 13.2 对性能瓶颈的统一认识

首版性能分析重点关注：

- DDR burst 组织
- 行缓存 / 小图缓存命中
- 双线性乘加流水线
- 坐标生成是否可递推

不把“三角函数开销”当作主要优化目标。
