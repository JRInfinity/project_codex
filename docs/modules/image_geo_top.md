# image_geo_top

> 依据文件：``rtl/top/image_geo_top.sv``。文档结论来自源码、现有文档和可追溯文件名；不能确定处标为“待确认”。

## 1. 模块定位
- 系统顶层，连接 AXI-Lite 控制域、AXI DDR 访问域和图像算法 ``core_clk`` 域。
- 顶层实例化 ``reset_sync``、``frame_config_cdc``、``cache_stats_cdc``、``scaler_ctrl``、``src_tile_cache``、``ddr_read_engine``、``rotate_core_bilinear``、``row_out_buffer``、``ddr_write_engine`` 和 ``taxi_axi_if``。
- 软件视角的一次任务由 AXI-Lite 配置和启动，硬件内部完成 DDR 读、cache/sample、几何算法、行缓冲、DDR 写回和中断上报。

## 2. 文件路径
- ``rtl/top/image_geo_top.sv``

## 3. 主要功能
- 提供 ``REG_CTRL_ADDR``、源/目标 base、stride、size、status、``REG_ROT_SIN_ADDR``、``REG_ROT_COS_ADDR`` 等寄存器。
- 提供 cache 旧统计 ``0x028`` 到 ``0x034``、``REG_CACHE_CTRL_ADDR``、扩展统计基址 ``0x040``、scheduler 控制 ``0x140`` 到 ``0x154``。
- 通过 ``frame_config_cdc`` 将控制域配置送入 core 域，通过 ``cache_stats_cdc`` 将统计快照返回 AXI-Lite 域。
- ``irq`` 由 ``reg_irq_en`` 与 ``done_sticky_reg``/``error_sticky_reg`` 组合产生。

## 7. 内部结构
- AXI-Lite 写路径根据地址更新配置寄存器，读路径返回配置、状态、统计和 scheduler 状态。
- ``frame_config_cdc`` 打包地址、stride、尺寸、旋转参数和 cache/scheduler 参数。
- 主数据流为 ``src_tile_cache`` -> ``ddr_read_engine`` -> ``rotate_core_bilinear`` -> ``row_out_buffer`` -> ``ddr_write_engine``。
- sticky done/error 让软件不会漏掉短脉冲，直到按寄存器语义清除。

## 8. 上游/下游连接关系
- 上游是软件 AXI-Lite master。
- 下游是 DDR AXI 从设备和 core 域各处理模块。
- ``taxi_axi_if`` 仅作为 AXI 接口桥接/引用，不在本项目逐模块展开。

## 9. 握手协议说明
- AXI-Lite 使用标准 AW/W/B 与 AR/R 通道握手。
- 配置提交依赖 ``cfg_valid_src/cfg_ready_src`` 到 ``cfg_valid_dst/cfg_ready_dst``。
- sample/read/writeback 分别使用 valid/ready 或 start/ready，顶层不应跨域组合观察内部 busy。
- ``irq`` 是电平式派生信号，软件清 sticky 后撤销。

## 10. 错误处理与边界条件
- AXI/DDR 错误由读写 engine 汇总为 task error，进入顶层 error sticky。
- cache 错误通过 ``src_tile_cache.error`` 参与任务错误路径。
- 寄存器非法组合是否全部硬件拦截需结合 ``scaler_ctrl`` 和子模块；不能从顶层端口名推断。

## 11. 综合/时序/CDC注意事项
- 顶层关键风险主要来自跨域边界和大扇出配置/统计路径。
- AXI-Lite、AXI、core 三域之间必须通过 CDC 模块或同步复位交互。
- 源文件存在注释乱码，应列为待清理项，不直接改 RTL。

## 12. 维护建议
- 改寄存器地址时同步更新软件驱动、testbench 和文档。
- 新增 cache/scheduler 字段时同步 ``frame_config_cdc`` payload 和 ``cache_stats_cdc`` 统计宽度。

## 13. 待确认问题
- 待确认：``REG_CTRL_ADDR`` 每一位的软件协议是否已有最终版。
- 待确认：最终是否同时保留 scale-only 与 rotate 两种顶层算法模式。
