# axi_burst_reader

> 依据文件：``rtl/axi/axi_burst_reader.sv``。文档结论来自源码、现有文档和可追溯文件名；不能确定处标为“待确认”。

## 1. 模块定位
- 位于 ``ddr_read_engine`` 的 AXI 侧，上游接收线性读任务，下游发 AXI ``AR`` 并消费 ``R`` 数据。
- 设计重点是把字节级任务拆成合法 AXI INCR burst，同时控制在途 burst/beat 数。

## 2. 文件路径
- ``rtl/axi/axi_burst_reader.sv``

## 3. 主要功能
- 按 ``BURST_MAX_LEN``、4KB 边界、``MAX_OUTSTANDING_BURSTS``、``MAX_OUTSTANDING_BEATS`` 拆分任务。
- 检查 ``RRESP``、``RLAST`` 和后级 backpressure，输出 ``result_done``/``result_error``。
- 使用 ``ddr_axi_pkg`` 的地址/长度计算函数。

## 8. 上游/下游连接关系
- 上游为 ``ddr_read_engine`` axi 域行任务。
- 下游真实 AXI 读通道通过顶层 ``taxi_axi_if`` 暴露。
- 输出 word 进入异步 FIFO，再到 ``pixel_unpacker``。

## 9. 握手协议说明
- ``task_ready`` 仅在 ``S_IDLE`` 且无 pending result 时为真。
- ``ar_fire = arvalid && arready``，发出 AR 后增加 outstanding 计数。
- ``r_fire = rvalid && rready``；``word_last`` 表示整个任务最后一个 word，不等于每个 burst 的 ``rlast``。
- ``result_valid`` 保持到 ``result_ready``。

## 10. 错误处理与边界条件
- ``rresp != OKAY``、``rlast`` 提前/延后、``r_fire && !word_ready`` 都会锁存错误或触发仿真检查。
- 源码断言检查 ``ARLEN``、4KB crossing、AR stalled 稳定性和 outstanding 溢出。

## 11. 综合/时序/CDC注意事项
- burst 规划与 outstanding gating 是主要组合路径，源码采用多级规划信号降低压力。
- FIFO 深度必须覆盖最大在途 beat，否则 R 通道可能在后级阻塞时溢出。
- 本模块无 CDC。

## 12. 维护建议
- 调整 outstanding 或 burst 上限时同步检查 FIFO 深度和 DDR 控制器能力。
- 4KB 拆分规则应保持与 writer/package 一致。

## 13. 待确认问题
- 待确认：目标 DDR 控制器对 outstanding 的实际接受能力。
